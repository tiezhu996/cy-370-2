## 项目主要需求

**项目概述**：图书馆座位预约管理系统 —— 面向高校图书馆，提供座位可视化预约、签到管理和学习小组研讨间预约的智能化系统。

**项目类型**：全栈Web应用

**功能需求**：
1. **楼层座位图可视化**：以楼层平面图展示所有座位分布，区分普通座位、靠窗座位、带电源座位，用颜色标识空闲/已约/使用中状态。
2. **按时段预约与选座**：用户选择日期和时段（最小单位30分钟或1小时），在座位图上点选空闲座位，系统自动校验时段冲突，预约成功生成二维码。
3. **扫码签到与违规释放**：预约开始前后15分钟内扫码签到，超时未签到自动释放座位并标记违约，使用结束需扫码签退，记录实际使用时长。
4. **座位偏好记忆**：系统自动记录用户历史预约偏好（如常选楼层、靠窗座位），下次预约时优先推荐相似座位，提升预约效率。
5. **学习小组研讨间预约**：提供独立研讨间（4-8人）预约功能，发起人邀请组员加入，预约成功后所有组员收到通知，研讨间配备白板和投影设备记录。

**技术要求**：
- 英文短名（用于 Docker Compose）：`ldlibseat`
- 前端端口：`28510`，后端端口：`29510`
- 前端框架：Vue 3 + TypeScript
- UI 库：Element Plus
- 构建工具：Vite
- 后端框架：Spring Boot + Java
- 数据库：MySQL 8.0
- 认证方式：JWT
- 其他依赖：JPA（Hibernate）、Maven

后端代码放在 `backend/`：

## 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

需要提供以下文件：
- 根目录 `docker-compose.yml`：编排前端、后端、数据库及所有依赖服务
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=ldlibseat`，确保 Compose 项目名不依赖目录名（中文目录名会被规范化为空导致 `project name must not be empty` 错误）
- `frontend/Dockerfile`：前端项目构建与运行镜像（多阶段构建，先构建再用 Nginx 托管）
- `backend/Dockerfile`：后端项目构建与运行镜像
- `frontend/nginx.conf`：Nginx 配置，处理前端路由和 API 反向代理
- `.env.example`：所有环境变量的示例文件，附注释说明；同时包含 `COMPOSE_PROJECT_NAME` 默认值

docker-compose.yml 要求：
- **不要写 `version:` 字段**（Compose V2 已废弃，会产生 `the attribute version is obsolete` 警告）
- 顶层声明 `name: ldlibseat` 作为兜底，避免目录名为中文时项目名为空
- 所有服务使用 `container_name` 命名容器，便于管理；容器名建议带 `${COMPOSE_PROJECT_NAME:-ldlibseat}` 前缀
- 数据库数据使用命名卷持久化（`volumes:` 顶层声明，不要使用绑定挂载到中文路径）
- 前端通过 Nginx 反向代理后端 API（前端代码里调用 `/api`，由 Nginx 转发到 `backend:29510`，不要硬编码 `localhost`）
- 服务间通过 Docker 内部网络通信，服务名即主机名
- 数据库服务配置 `healthcheck`，后端服务通过 `depends_on: { db: { condition: service_healthy } }` 等待数据库就绪
- 后端服务也建议配置 `healthcheck`，前端 `depends_on` 后端健康
- 暴露端口：前端 `80:80`，后端按框架默认端口，数据库使用标准端口
- 所有可变配置（端口、数据库账号密码、JWT 密钥等）通过环境变量注入，并在 `.env.example` 中给出默认值
- 支持通过 `docker compose up -d` 一条命令启动全部服务，不依赖额外的 `-p` 参数

`.env` / `.env.example` 必须包含的键：
- `COMPOSE_PROJECT_NAME`：英文短名（小写字母/数字/`-`/`_`），如 `ldlibseat`
- 数据库连接相关：`DB_NAME` / `DB_USER` / `DB_PASSWORD` / `DB_ROOT_PASSWORD`
- 后端：`JWT_SECRET` 或等价的密钥配置
- 端口映射：`FRONTEND_PORT`、`BACKEND_PORT`、`DB_PORT`（可选，便于本地冲突时调整）

Nginx 反向代理要求（`frontend/nginx.conf`）：
- `location /api/` 反向代理到 `http://backend:29510/`
- 配置 `try_files $uri $uri/ /index.html;` 支持前端 SPA 路由
- WebSocket 场景需配置 `Upgrade` / `Connection` 头

自检清单（生成代码后必须确认）：
1. 在 **中文目录名** 下执行 `docker compose config --quiet` 不报错
2. 启动后 `docker compose ps` 显示全部容器 healthy
3. 浏览器访问前端端口能看到页面，且 `/api/...` 请求返回正常
4. README 中的 Docker 启动命令是 `docker compose up -d`（不需要 `-p`）

## docker-compose.yml 参考骨架（务必遵循）

```yaml
name: ldlibseat   # 顶层 name 字段兜底，避免中文目录名导致 project name 为空

services:
  db:
    image: mysql:8.0
    container_name: ${COMPOSE_PROJECT_NAME:-ldlibseat}-db
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
      MYSQL_DATABASE: ${DB_NAME}
      MYSQL_USER: ${DB_USER}
      MYSQL_PASSWORD: ${DB_PASSWORD}
    volumes:
      - db_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "root", "-p${DB_ROOT_PASSWORD}"]
      interval: 5s
      timeout: 5s
      retries: 20

  backend:
    build: ./backend
    container_name: ${COMPOSE_PROJECT_NAME:-ldlibseat}-backend
    restart: unless-stopped
    environment:
      DB_HOST: db
      DB_PORT: 3306
      DB_NAME: ${DB_NAME}
      DB_USER: ${DB_USER}
      DB_PASSWORD: ${DB_PASSWORD}
      JWT_SECRET: ${JWT_SECRET}
    depends_on:
      db:
        condition: service_healthy

  frontend:
    build: ./frontend
    container_name: ${COMPOSE_PROJECT_NAME:-ldlibseat}-frontend
    restart: unless-stopped
    ports:
      - "${FRONTEND_PORT:-80}:80"
    depends_on:
      - backend

volumes:
  db_data:
```
